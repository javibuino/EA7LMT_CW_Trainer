IAMBIC B mode

X = dit  |  Z = dah

Hold down each key to send repeated elements. If you press both at once (squeeze), the keyer alternates dit/dah automatically, just like a real electronic keyer in Iambic B mode: if both keys are released while an element is being sent, one extra alternated element is completed before stopping. Each element's duration is set by the keyer according to the WPM setting.

VERTICAL mode (straight key)

X or space = the key. There's no electronic keyer here: you control each element's duration by holding the key down. A short tap is decoded as a dot and a long one as a dash, using the dit duration at the configured WPM as the reference (any press longer than twice that duration is read as a dash). This mode trains your own hand timing, just like a real straight key.

Speed is measured in WPM (words per minute, PARIS standard). In iambic mode it controls each element's duration; in vertical mode it's only used as a reference to decode your keystrokes.

Purple: the box at the top shows, in purple, the pattern (dots/dashes) of the letter you're keying right now, not yet resolved.

Yellow: in the text line, letters that have already been decoded and confirmed (after the letter gap).

Green: within that line, the most recent letter you just typed, so you can spot it at a glance.

Cyan: a decoded prosign (like <BT> or <SK>) — sent as two letters with no pause between them. See the REF table for the full list.

CLR clears the decoded text.
